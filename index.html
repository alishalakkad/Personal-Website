PROJECTS

LettuceCOOK
This was team project which wedeveloped over the course of 3 months. It's an Android Application that suggests receipes using the Spoonacular API to households based on the ingredients present in their house. It does so by letting the user keep a track of their stock and grocery lists. The inspiration behind this app was the life of a college student. Anyone who lives such a fast paced life doesn'thave the time to think of what they can make for lunchor dinner. This way they can keep tabs on their grocery and see what recipes they can make with the ingedients available. We used Firebase and Firestore for the backend and server of this app. Some features of the application include:
  • Create a household and invite other users to it
  • Add grocery items to stock or grocery list and able to move them from onelist to the other
  • Be able to send friend requests to other households and accept any friend requests
  • Select ingredient and ask the app to suggest a recipe and then list the missing ingredients by comparing your stock and the ingredients of the recipe
  • Ask friend household for ingredients or add the missing ingredient to the grocery list
  • Save recipes to view later or share recipes with other households
  • Invite friend households for dinner
  • Be able to look up near by restaurants and grocery stores using the Google Maps API



Plan-it
This was also a group project aimed to create a website that makes event managementand task assignment easy. When an event is created by a user, a unique randomized event code is created and has to be used for other users to join that event. Then, people in that event can create tasks and assign them to a person in that event. Once the tasks are completed, tasks can be marked as completed and they are added to a "Completed" list. My role in this project was to set up MongoDB Atlas to store any information entered by the user, write backend functions and server side code in Node.js and make sure the task are printed correctly and updated in real-time. This website can be viewed here

Lyrico
I wanted to make this project because I had never programmed in Swift and I wanted explore a new language. I began making this project in a Hackathon and tried to learn this language from scratch. Over the course of the Hackthon my team and I developed an iOS game where the user has to match the song to its corresponding lyric. We had stored all the songs in a dictionary and displayed the correct option and 3 other randomized options. The app played an appropriate sound if the user got the question right or wrong. The user could also exit the game and view their final score during time part in the game.

Water-Tracker
This project was mainly aimed toward a problem I face myself. I barely stay hydrated and wanted to make an application that can help me track my water intake. Water-Tracker accounts for many brands of water bottles like Dasani and SmartWater and has a slider to indicate how much water I drink and calculates it in fl. oz. accordingly.

Other Projects
Apart from these projects, I have also done projects which are a part of our class curriculum such as:
  1. Shell - I created my own shell in C and C++ to execute commands such as ls, cd, grep, etc. LEX and YACC was used to parse any commands input by the user. I also implemented wildcards, subshell and pipes for the execution of my shell.
  2. Malloc implementation - I wrote a malloc implementation in C and C++ to allocate memory and deallocate memory in a heap and call sbrk() to get a new block of memory when more was needed. We had to coalesce any adjacent free blocks and insert all unallocated blocks into a double linked list.
  3. Hash Map and Quadratic Probing - We implemented the concepts of hash maping and quadratic probing in Java, to insert, traverse through and manipulate data.
  4. Family Tree - In this project we were required to create a binary tree and create a family tree based on the data given. We also had to use algorithms such as BFS and DFS to find the closest common ancestors between two family members.
  5. Hardware and coding in Assembly - We made a traffic light using decoders, capacitors, resistors, etc on a breadboard. We implemented fibonacci, insertion sort and substring in Assembly. We also coded the printf function in Assembly and C.
